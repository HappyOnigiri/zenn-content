# Project Rules

## プロジェクト概要

このプロジェクトは Zenn (https://zenn.dev) の記事管理リポジトリです。
`zenn-cli` を使用してコンテンツを管理します。

### 自動コミット禁止

**ユーザーからの明確な指示（例：「コミットしてください」）がない限り、自動で Git コミットを行わないでください。**

ファイル名は `zenn-cli` が生成するデフォルトのランダムな文字列を維持することを推奨します。

## 記事執筆ルール

### 基本方針

- 「です・ます」調で統一する
- 一文は長くても80文字程度を目安にする
- 一文は句点（。）のタイミングで改行する
- 結論や要点を先に書き、詳細は後に続ける

### Zenn のマークダウン記法

- Zenn 独自記法を正しく使うこと
- メッセージボックス: `:::message` / `:::message alert`
- アコーディオン: `:::details タイトル`
- コードブロックにはファイル名を付ける: ` ```js:index.js `
- 数式は KaTeX 記法: `$inline$` / `$$block$$`
- リンクカードは URL を単独行に置くだけで生成される

### 構成

- frontmatter（title, emoji, type, topics, published）を必ず含める
  - topics は最大5つまで（Zenn の制約）
  - topics はすべて小文字で書く（例: "nextjs" ではなく "Next.js" でもなく "nextjs"）
- 見出しは h2（##）から始める（h1 は title が担う）
- 長い記事は適度に h2 / h3 で区切る
- コード例は実際に動くものを書く。省略する場合は明示する

### やってはいけないこと

- `published: true` を勝手に変更しない

## 用語・表記の統一

### 固有名詞（正しい表記で書くこと）

- GitHub（× Github, github）
- VS Code（× VSCode, vscode）
- TypeScript（× Typescript）
- Next.js（× Nextjs, next.js）
- Docker（× docker ※文中での一般名詞的用法を除く）
- Zenn（× zenn）
- Cursor（× cursor ※エディタを指す場合）

## 画像管理ルール

Zenn の画像管理は `/images` 配下のサブディレクトリで行います。

### 画像の配置

- すべての画像ファイルは `/images/<記事のslug>/` 配下に保存してください。
- ファイル名は「何を示しているか」がわかる具体的な名称にする（例: `main-play-screen.png`, `architecture-diagram.png`）。
- スクリーンショットのデフォルト名（例: `2026-02-20 3.32.58.png`）や、意味のない文字列のまま保存しない。
- `/images` 直下に直接ファイルを置かないでください。

### 画像の参照方法

Markdown 内で画像を参照する際は、絶対パス形式（`/images/<slug>/filename.png`）を使用してください。

```markdown
![代替テキスト](/images/<slug>/filename.png)
```

### 画像の制約

- 画像ファイルのサイズが 3MB 以下であること。

### 画像の挿入

- 画像を挿入する場合は alt テキストを必ず書く。
- スクリーンショットは手順の前後関係がわかるように配置する。
- ユーザーが画像をアップロードまたは追加しようとしている場合、`/images/<記事のslug>/` ディレクトリへの配置と、**内容に基づいた具体的なファイル名へのリネーム**を提案してください。
- 既存の記事にある画像パスやファイル名がこの構成に従っているか確認してください。
