## はじめに

以前、こんな記事を書きました。

https://zenn.dev/happy_onigiri/articles/a94ae67c605ad5

「Cursor を月500ドル使う開発者が本気で Antigravity を2週間使った感想」として、Antigravity をかなりポジティブに評価しました。
個人開発は Antigravity へ完全移行したとまで宣言しています。

あれから数ヶ月。
私と Antigravity の睦まじい関係は、突然終わりを迎えました。

これは、一人の開発者と AI IDE の出会いから別れ、そして新しいパートナーとの出会いを綴った記録です。

## Antigravity との幸せな日々

前回の記事以降も、個人開発では Antigravity をメインに使い続けていました。
ファミリーグループによるクォータ運用も安定しており、Gemini 3.1 Pro をメインに、Flash をサブとして使い分ける日々。
コストを気にせず開発できるのは、やはり快適でした。

一時期は Antigravity を3つ同時に起動して並行開発を進めていたこともあります。
三股と言われても否定できません。それくらい、熱量は本物でした。

この期間に、いくつかの個人プロジェクトを Antigravity で開発しました。

- **[メスガキポンポン](https://onigiri-game-portal.vercel.app/mesugaki-pong/)**: 煽られながらプレイするカオスなピンポンゲーム
- **[量子マグロ亭](https://onigiri-game-portal.vercel.app/quantum-maguro/)**: 回転寿司をテーマにした爽快なタイピングゲーム

私たちの関係は順調そのものに見えていました。

## クォータ制限の突然の厳格化

ある日を境に、明らかに様子がおかしくなりました。

Gemini 3.1 Pro を2〜3時間使っただけで、クォータのリセットに **一週間** かかるようになったのです。

![クォータのリセットまで5日と表示されている画面](/images/6ca9f26c08d1be/quota-reset-5days.png)

以前は5時間サイクルで回復していたクォータが、ある時期を境に週次ロックの閾値が大幅に引き下げられました。
私だけの問題かと思い調べてみると、同様の報告が複数見つかりました。

https://note.com/major_elk2890/n/n41b2c449694b

https://www.reddit.com/r/google_antigravity/comments/1riww8t/why_do_gemini_31_quotas_take_longer_to_reset_than/

どうやら私だけが振られたわけではなかったようです。

### 「Pro」とは何だったのか

Google AI Pro は月額の有料プランです。
にもかかわらず、メインモデルである Gemini 3.1 Pro が実質的に週2〜3時間しか使えない。
これではまともに開発を継続できません。

Gemini 3.1 Flash は従来通り十分なクォータが用意されています。
しかし、Flash だけで常に開発するのは現実的ではありません。
複雑なロジックの実装やバグの特定など、Pro モデルの精度が必要な場面は日常的に発生します。

以前の記事で紹介したファミリーグループを活用しても、もはや焼け石に水でした。
全アカウントが同じペースでクォータを消費するため、1日で全滅します。

## 別れの決断

クォータが枯渇するたびに開発が止まる。
一週間待ってようやく回復しても、また数時間で使い切る。

開発ツールに求めるものは、最先端の機能だけではありません。
「使いたいときに使える」という当たり前の安定性です。

私はついに、Antigravity との別れを決断しました。

## 新しいパートナー：Claude Code

別れた直後に新しいパートナーと付き合い始めるのは少々気まずいですが、正直に書きます。

気づけば、Claude Max 5x プランに課金していました。

### Claude Max 5x プラン

月額 $100。安くはありません。
しかし、執筆時点（2026年3月）ではプロモーション期間中で利用量が2倍になっており、ハイエンドモデルの Opus をガンガン使っても5時間のクォータを半分も使い切れない状態です。

https://support.claude.com/en/articles/14063676-claude-march-2026-usage-promotion

Antigravity で頭を悩ませていたクォータ問題が、嘘のように消えました。

### Claude Code との出会い

Claude Code はターミナルベースの AI コーディングツールです。
IDE に縛られず、ターミナルから直接 Claude と対話しながら開発を進められます。

正直なところ、見た目は地味でした。
VSCode ライクな IDE に慣れきった身としては、ファイルツリーもなければ GUI のボタンもない、飾り気のないターミナル画面。
第一印象は「え、この人…？」という感じです。

でも、付き合ってみると印象はガラリと変わりました。

### この人のどこが好きか

**約束を守る人だった。** Antigravity との日々で慣れてしまった「突然つながらなくなる」というストレスが、Claude Code では一切ありません。地味だけど、毎日一緒にいるなら、派手さより信頼感です。

**最初から本気で向き合ってくれる。** Antigravity では「Gemini で実装 → バグ → Claude Opus で修正したいけどクォータが…」というジレンマがありました。Claude Code では最初から Opus で設計・実装できるので、手戻りが格段に少ない。最初のデートから全力で来てくれる人は、やっぱり信頼できます。

**私より先に問題に気づいてくれる。** これが一番驚いた点です。指示を出す時点で私が気づいていなかった課題を、プラン段階で見つけて相談してくれます。「ここはこういう問題が起きそうですが、どうしますか？」と。こちらの意図を汲んだうえで先回りしてくれる、そんなパートナーです。

### もちろん完璧ではない

ブラウザの自動操作は Claude Code にはできません。UI の確認は手動で行う必要があります。
Antigravity ではデザインの生成からブラウザでの検証まで一気通貫でやってくれていたので、この点では元カノのほうが器用だったと認めざるを得ません。

実際、個人サイト（[onigiri-portal](https://onigiri-portal.vercel.app/)）の実装は Claude Code で行いましたが、デザインだけは限られたクォータを使って Antigravity の Gemini 3.1 Pro に作ってもらいました。
別れた恋人にときどき会いに行くような、少し複雑な気持ちです。

## おわりに

Antigravity は良いツールでした。今でもそう思っています。
ブラウザの自律操作は唯一無二の強みだし、サブスクリプションモデルのコンセプトも素晴らしい。

ただ、有料プランでありながら予告なくクォータが大幅に制限されてしまうと、メインの開発ツールとして信頼を寄せ続けるのは難しかった。

もしクォータの制限が緩和される日が来たら、またあの頃のように並行開発を楽しめるかもしれません。
でも今の私には、新しいパートナーがいます。

……たまにデザインだけお願いしに行くけど、それはまた別の話。

今、私は幸せです。
