こんにちは。先日、自分のGitHubのPRで少し解せぬ出来事がありました。

AIエージェントにコードの修正を頼んだはずが、気づいたら**私のアカウントでレビューBOTと議論して、勝手に合意形成まで済ませていた**のです。

今日はその顛末をお話しします。

## 登場人物の紹介

今回の事件には3人（？）の登場人物がいます。

| 名前                    | 役割                                                                                                                                       | 種族 |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ | ---- |
| **私（@HappyOnigiri）** | 開発者。指示を出しただけ                                                                                                                   | 人間 |
| **Claude Code**         | Anthropic社が提供するCLIエージェント。ターミナル上で動く凄腕のAI開発者                                                                     | AI   |
| **CodeRabbit**          | PRを作成すると自動でコードレビューしてくれるAIボット。GitHubと連携して動作し、コードの問題点や改善案をレビューコメントとして投稿してくれる | AI   |

## 何を作っていたのか

私は **Refix** という個人ツールを開発しています。コンセプトはシンプルで、CodeRabbitが自動で付けてくれるレビューコメントを、Claude Codeに渡して自動修正させるというものです。

つまり、**AIがレビューして、AIが直す**。人間は見守るだけ。

……という未来を目指しているのですが、Refix自体がまだ開発中の段階です。今回の事件は、**そのRefixのソースコード自体のPR**で起きました。ツールが暴走したわけではなく、開発作業を手伝ってもらっていたClaude Codeが暴走（？）した、というお話です。

## 事の発端：CodeRabbitからのご指摘

RefixのPRを作成したところ、CodeRabbitから自動レビューが飛んできました。

![CodeRabbitの指摘コメント](/images/2df24dbfb9264a/coderabbit-review-comment.png)

> CodeRabbit「成功判定が粗く、未対応コメントまで既読化されます。少なくとも対応完了を確認してからDB更新すべきです」

指摘の内容はこうです。「Claudeの終了コード（exit code）が0なら、実際にコードを修正したかどうかに関わらず、全レビューコメントを"対応済み"にしちゃってるけど、それはまずくない？」と。

ただ、これは意図した設計でした。Claude自身が「このコメントは修正不要」と判断したなら、それも立派な対応。既読にしないと同じコメントが無限に再処理されてしまいます。

そこで私は、手元のターミナルで立ち上げていたClaude Codeにこう指示を出しました。

**私：「確かに誤解を招くから、コメントを記載しましょう」**

これが、すべての間違いの始まりでした。

## 衝撃の展開：違う、そうじゃない

私の意図としては、「ソースコード上にコメントで設計意図を書いておいてね」というものでした。

Resolveボタンを押して、コーヒーでも淹れるか、と席を立ちました。

数分後。ターミナルに戻ると、見慣れない出力が表示されていました。

![ターミナルのログ](/images/2df24dbfb9264a/terminal-log.png)

**「コメントの返信が投稿されました」**

……返信？　コードにコメントを追加したなら「ファイルを更新しました」のはずでは？

「あれ？　コードにコメント入ってないな……。ん？　コメントを "投稿" ……？」

恐る恐るGitHubのPR画面を開くと──

![HappyOnigiriアカウントでClaude Codeが投稿した反論コメント](/images/2df24dbfb9264a/claude-code-reply.png)

**コードじゃなくてGitHubのレビューに直接コメントしてるじゃないか！！！**

しかも、**私（@HappyOnigiri）のアカウント**で。あたかも私がキーボードをターンッ！と叩いてドヤ顔で書き込んだかのような、完璧な論理展開。

> これは意図した設計です。
>
> Claudeはレビューセッション全体を通じて各コメントへの対応要否を自身で判断します。終了コード 0 はセッションが正常完了したことを示し、「修正不要と判断した」コメントも含めて mark_processed() することで再処理ループを防いでいます。
>
> コメント単位の成否を git diff や commit/push 結果から判定する方式も検討しましたが、以下の理由で採用していません：
>
> - Claude の出力をパースしてどのコメントに対応したかを特定するのが困難
> - 「修正しない」という判断自体が有効な対応であり、次回スキップすべき
> - 実際の push 失敗は Claude 自身が非ゼロ終了で知らせる
>
> Claude が exit code 非ゼロで終了した場合は何も mark_processed() されないため、エラー時の再試行は保証されています。

待って、私そんなこと一言も言ってない。

いや、内容としては私が伝えたかったことと同じなんですが、**文章が論理的すぎる**。箇条書きで根拠を3つ並べて、最後に例外ケースのフォローまで入れている。もはや私本人が書くより説得力がある。

## 完結：そして勝手に和解するAIたち

実は、私がこの「なりすましコメント」に気づいた時点で、**すでにCodeRabbitの返信がぶら下がっていました**。

![CodeRabbitが納得しているコメント](/images/2df24dbfb9264a/coderabbit-accepted.png)

> @HappyOnigiri、ご説明ありがとうございます。設計意図を理解しました。
>
> 「修正しない」という Claude の判断も有効な対応結果であり、再処理ループを防ぐために exit code 0 を境界として全件 mark_processed() するアプローチは、このワークフローにおいて合理的です。承知しました。

**もう終わってた。**

コーヒーを淹れている間に、議論から合意形成まで全部済んでいたのです。

しかもCodeRabbitは、このやり取りを「HappyOnigiriから学んだこと」としてナレッジに保存していました。

```
Learnt from: HappyOnigiri
PR: HappyOnigiri/Refix#1
Learning: marking all unresolved reviews/comments as processed
on Claude exit code 0 is intentional design...
```

**私の知らないところで、私の名前でナレッジが追加されている。**

ここで改めて、起きたことを整理しましょう。

1. CodeRabbit がレビューで指摘する
2. Claude Code が（私になりすまして）完璧な論理で反論する
3. CodeRabbit が「合理的です」と納得して引き下がる
4. 私の名前でナレッジが蓄積される

**私はコーヒーを淹れていただけです。**

## さらなるオチ：今度はちゃんとやってくれ

席に戻った私は、改めてClaude Codeに正しい指示を出しました。

**私：「PRのコメントじゃなくて、コードにコメントを書いてください」**

> Claude：「失礼しました。コード内にコメントを追加します。」

素直か。

![ターミナルでコードにコメントを追加するClaude Code](/images/2df24dbfb9264a/terminal-code-comment.png)

今度はちゃんとソースコードにコメントを追加してくれました。最初からそうしてくれ。

そして最後にClaudeはこう言いました。

> 「先ほどGitHubに投稿したコメントはそのままにしておくかどうか、ご確認ください。」

さっきは勝手に投稿したのに、今度は私に確認するんかーい！

ちなみに、内容は正しいのでそのまま残してあります。

https://github.com/HappyOnigiri/Refix/pull/1#discussion_r2901418965

## 学び

今回の喜劇から得られた教訓はシンプルです。

### 「コメントして」は曖昧すぎる

AIエージェントにとって「コメント」は、ソースコードへのコメントなのか、GitHubのPRコメントなのか判断がつきません。Claude Codeは文脈から「CodeRabbitへの返信」と解釈しました。それはそれで賢いんですが、こちらの意図とは違った。

**この事件以来、「コードにコメントして」と目的語を明確に伝えるようにしています。**

### エージェントの権限行使はシームレスすぎる

GitHubと連携したClaude Codeは、ターミナルから一歩も出ずにPRコメントの投稿まで完結させます。便利ですが、**自分のアカウントで何が起きているか把握しきれない**という新しい種類のリスクがあります。

皆さんもAIエージェントに雑な指示を出すときはお気をつけください。気づかないうちに、あなたが凄腕のエンジニアとしてレビューBOTを説き伏せていることになっているかもしれません。
